# 纵向细沟检测算法说明

## 主要函数

`detect_longitudinal_grooves` 用于检测 128×128 小图中的纵向细沟或纵向钢片。该函数属于 core 算法层，只输出检测特征，不接收小图类型，不进行打分，也不判断 center / side 的数量是否合规。

调用方可基于返回的纵向细沟数量、中心位置列表和宽度列表执行后续业务逻辑。

## 输入参数

- `image`：BGR 图像数组，形状必须为 `(H, W, 3)`。
- `nominal_width_px`：纵向细沟名义宽度，单位为像素。
- `min_width_px`：候选线段的最小逐行均值宽度，单位为像素。
- `max_width_px`：候选线段的最大逐行均值宽度，单位为像素。
- `narrow_cluster_px`：单行窄簇筛选宽度上限，单位为像素。
- `edge_margin_px`：左右边缘忽略宽度，单位为像素。
- `min_segment_length_px`：候选线段的最小纵向长度，单位为像素。
- `max_angle_deg`：相邻行中心点允许偏离竖直方向的最大角度。
- `is_debug`：是否返回调试图像和掩码。

## 输出结果

函数显式返回五元组，顺序如下：

- `groove_count`：检测到的纵向细沟数量。
- `groove_positions_px`：每条纵向细沟的中心列坐标，单位为像素。
- `groove_widths_px`：每条纵向细沟的逐行均值宽度，单位为像素。
- `line_mask`：仅在 `is_debug=True` 时返回纵向细沟掩码，否则为 `None`。
- `debug_image`：仅在 `is_debug=True` 时返回调试标注图，否则为 `None`。

## 算法流程

1. 将 BGR 图像转为灰度图，并进行 3×3 高斯模糊降低噪声。
2. 使用自适应阈值把暗色纵向细沟转换为白色前景。
3. 忽略左右边缘 `edge_margin_px` 范围内的像素，减少主沟裁切残留误检。
4. 使用纵向形态学闭运算桥接细沟中的小断点，再用纵向开运算去除零散噪声。
5. 对连通域逐行提取窄簇，并跨行追踪形成候选纵向轨迹。
6. 按局部偏转角度切分候选轨迹，保留接近竖直方向的连续子段。
7. 使用像素宽度和像素长度阈值过滤候选子段。
8. 对横向位置接近且纵向范围高度重叠的子段去重，输出最终数量、位置和宽度。

## 异常行为

函数不吞掉异常，也不把异常包装成字典返回。入参类型错误会抛出 `InputTypeError`，入参数据不满足算法前置条件会抛出 `InputDataError`。

## 调试行为

默认 `is_debug=False`，返回结果中的 `line_mask` 和 `debug_image` 均为 `None`。

当 `is_debug=True` 时，函数会返回：

- `line_mask`：白色像素表示检测到的纵向细沟区域。
- `debug_image`：在原图上叠加细沟掩码和中心线，用于人工检查检测结果。
